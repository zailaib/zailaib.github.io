# House Dismantle 3D — 设计审查报告

> 审查日期：2026-07-23 | 审查范围：全部 1867 行代码（9 个源文件）

## 总览

整体架构形似传统三开间民居（12m×9m，两层，坡屋顶，L 型楼梯），但几乎每个子系统之间都存在空间逻辑、结构尺度、视觉一致性的硬伤。以下按类别逐一列举，附代码定位和修复方向。

---

## 一、空间逻辑矛盾

### 1.1 后墙：窗户与后门重叠

**文件**: `house-openings.js:126-150`

后窗覆盖全部三个开间：
```js
// L126: 后窗 —— 三个开间都有
[-BAY_W, 0, BAY_W].forEach(x => { winGrp.add(makeWindow(x, winY, winZb, Math.PI)); });
// L131: 后门 —— 也在右开间
const backDoorX = BAY_W;
```

右开间（x=+4）**同时有后窗和后门**，物理上重叠。后墙本来还有个神龛紧贴。三者在同一面墙同一位置争夺空间。

**修复方向**: 后门移到左开间或中间开间；或者右开间取消后窗。

### 1.2 前后隔墙门洞 ≈ 10cm

**文件**: `house-core.js:105-115`

隔墙分三段，段间留缝作为门洞：
```js
const segW = BAY_W - WALL_T*0.4; // 4 - 0.096 = 3.904m
// 三段分别位于 x = -4, 0, 4
// 段间缝隙 ≈ WALL_T * 0.8 ≈ 0.19m ≈ 19cm
```

实际上是 `WALL_T * 0.8 ≈ 19cm`（之前计算 10cm 偏保守），但仍然远小于内门洞 0.9m 的实际需求。人无法正常通过。

**修复方向**: 段间留出至少 0.9m 的门洞，或改用门框网格（`doorway` 子网格）。

### 1.3 L 型楼梯顶部出口与二楼栏杆位置不匹配

**文件**: `house-openings.js:168-238` vs `house-core.js:134-146`

楼梯终点：
- Segment 2 最后一级在 `z = seg2StartZ + stepW + 8 * stepD ≈ -0.5 + 0.8 + 2.24 = 2.54`
- x 在 `seg2StartX + stepW/2 ≈ 0.44 + 0.4 = 0.84`

二楼栏杆围栏：
- `railX = 0.2, railW = 1.0` — 中点在 x=0.7
- `railZ = -0.5, railD = 1.2` — 中点在 z=-0.5

楼梯出口（z≈2.54）在栏杆围区（z≈-0.5 到 +0.1）之外约 2.4m。人到二楼后需要跨一大步才能进入栏杆保护区，且二楼没有楼梯口平台。

**修复方向**: 统一楼梯出口坐标与栏杆开口位置；在出口处加过渡平台。

## 二、结构逻辑问题

### 2.1 中心柱 —— 客厅正中立两根柱子

**文件**: `house-core.js:66-76`

```js
const colPositions = [
  // ...前后各三根（正常）
  [-HW2+0.3, 0], [HW2-0.3, 0],  // z=0 轴线上两根柱子
];
```

前后隔墙在 z=1.0，后墙在 z=-4.5，前墙在 z=+4.5。z=0 是进深正中间——**客厅/堂屋的正中央**。传统民居绝无在正厅中央立两根柱子的做法。木柱应紧贴隔墙（前后隔墙位置 z=1.0），而非远离隔墙 1m。

**修复方向**: 将 z=0 的两根柱子移到 `z = crossZ`（1.0）处，紧贴隔墙。

### 2.2 墙体厚度视觉单薄

**文件**: `config.js:12`

`WALL_T = 0.24`（24cm）数值合理，但在 12m×9m 的体量下：
- 外观看像纸板而非实体砖墙
- BoxGeometry 无倒角，边缘锐利
- 内墙仅 `WALL_T * 0.8 = 19cm`，与外墙视觉差异小

**修复方向**: 考虑用 `RoundedBoxGeometry` 或给墙体加 0.02-0.05 的边缘微调；内墙用更薄的独立尺寸。

### 2.3 二楼山墙高度异常

**文件**: `house-core.js:184-185`

```js
makeUpperWall('upperWallLeft', '山墙(左)', ...,
  WALL_T, WALL_H2 + 0.5, HOUSE_D,   // h = 1.5 + 0.5 = 2.0m
  -HW2, WY2 - 0.25, 0, ...);        // y = WY2 - 0.25
```

山墙比前后墙高 0.5m，但位置却下移了 0.25m。净效果是顶部只比前后墙高 0.25m。山墙应该**向上延伸**填充屋顶三角区域（2.8m 处到 5.5m 屋脊），而非往下移。

**修复方向**: 山墙应使用梯形几何体或从 BAND_Y 直接延伸到 ROOF_H，尺寸应随 x 变化。

## 三、视觉渲染问题

### 3.1 材质缺乏纹理

**文件**: `config.js:28-70`

所有材质均为纯色 `MeshStandardMaterial`，无任何纹理贴图：
- 瓦片屋顶（`roofTile: 0x4a4a5a`）→ 看起来像光滑塑料
- 木柱/木梁（`column: 0x6b3a20`）→ 无法与木地板区分
- 石基（`base: 0x6e6e6e`）→ 没有石块接缝

**修复方向**: 最低成本方案——用 `canvas` 生成程序化纹理贴图（木纹、瓦纹、石纹），设为各材质的 `map` 属性。

### 3.2 通风窗 Z-fighting

**文件**: `house-core.js:163-181`

```js
const frame = box(0.5, 0.5, 0.06, MATS.window);
frame.position.set(vx, WY2, z);          // z = ±(HD2 + WALL_T/2 + 0.03)
const mullH = box(0.4, 0.04, 0.07, MATS.window);
mullH.position.set(vx, WY2, z);          // 完全相同 z
const mullV = box(0.04, 0.4, 0.07, MATS.window);
mullV.position.set(vx, WY2, z);          // 完全相同 z
```

窗框、横梃、竖梃三层 mesh 在**完全相同深度**，渲染时产生 Z-fighting 闪烁。

**修复方向**: 横梃 z 偏移 +0.01，竖梃 z 偏移 -0.01。

### 3.3 低多边形物体与其他部分精度不一致

**文件**: `house-yard.js`, `house-interior.js`

| 物体 | 几何体 | 分段数 | 视觉效果 |
|------|--------|--------|----------|
| 木柱 | CylinderGeometry | 16 | 圆滑 |
| Wok | SphereGeometry | 16×8 | 圆滑 |
| 鸡身 | SphereGeometry | 8×6 | 棱角分明 |
| 鸡冠 | ConeGeometry | 4 | 方块状 |
| 井顶 | ConeGeometry | 4 | 金字塔状 |
| 井壁 | CylinderGeometry | 16×1 | 管状（无顶部） |

低多边形风格本身没问题，但不能一半精细一半粗糙——视觉上不统一。

**修复方向**: 要么全部升到 12-16 段，要么统一降到 6-8 段做 stylized low-poly 风格。

### 3.4 井壁无顶底

**文件**: `house-yard.js:86-89`

```js
const wellWallGeo = new THREE.CylinderGeometry(0.38, 0.42, 0.6, 16, 1, true);
// openEnded = true — 井壁是个开放的管道
```

第 5 个参数 `openEnded = true` 意味着井壁没有顶面和底面。从某些角度看会看到空心管道内部。

**修复方向**: 设 `openEnded = false`，或加独立的顶/底环。

## 四、比例与尺度问题

### 4.1 二楼墙体过矮

**文件**: `config.js:9`

`WALL_H2 = 1.5` —— 二楼墙体净高仅 1.5m。扣除地板厚度后，可使用高度约 1.3m。传统民居二层（阁楼/半层）至少需要 1.8-2m 才可正常使用。

**修复方向**: `WALL_H2` 至少设为 1.8-2.0m，相应调整 `ROOF_H` 以保持屋顶坡度。

### 4.2 桌椅间距过紧

**文件**: `house-interior.js:96-113`

桌子 2.0m×1.5m，四条长凳距桌边仅约 0.1m（桌子 edge 到凳子 center）：
```js
// 桌子边缘在 z = 2.5 ± 0.75 = (1.75, 3.25)
// 前凳 z = 1.4（距桌边 0.35m）
// 后凳 z = 3.6（距桌边 0.35m）
// 左凳 x = -1.1（距桌边 0.1m）
// 右凳 x = 1.1（距桌边 0.1m）
```

前后凳间距尚可，左右凳距桌边太近。且凳子宽 1.6m 而桌面深 1.5m——凳子比桌子还长。

**修复方向**: 左右凳外移 0.2-0.3m；凳子长度缩短到 1.2m。

### 4.3 床间距过窄

**文件**: `house-interior.js:67-69`

两张床沿 Z 轴排列：
```js
bedGrp.add(makeBed(0, -HD2 + 1.5)); // z = -3.0
bedGrp.add(makeBed(0, -HD2 + 3.5)); // z = -1.0
```

床长 1.6m（含 headboard 0.8m 延伸），两张床间距 2m。扣除床身后通道不足 0.4m。

**修复方向**: 两张床并列（沿 X 轴）而非前后排列；或增大间距。

## 五、拆解逻辑问题

### 5.1 非自然的反向依赖

**文件**: `config.js:75-119`

```js
{ name: 'stairs',   deps: ['upperWallFront', 'upperWallBack'] },
{ name: 'crossWall', deps: ['upperWallFront','upperWallBack','interiorWall1','interiorWall2'] },
{ name: 'pipelines', deps: ['base'] },
```

- **楼梯依赖二楼墙**：必须先拆二楼前后墙才能拆楼梯。现实中楼梯是独立结构
- **一层隔墙依赖二层墙**：一层的前后隔墙竟依赖二层外墙，结构上毫无道理
- **排水管依赖石基**：排水管在地基外围，应先于石基拆除

**修复方向**: 改为实际的施工依赖——按自顶向下、自外向内、自软装到硬结构的顺序重新设定依赖关系。

## 六、代码一致性问题

### 6.1 前门位置注释与代码矛盾

**文件**: `house-openings.js:27-30`

```js
// FRONT DOOR (left bay, at x = -BAY_W)   ← 注释说左开间
const doorX = BAY_W; // right bay center   ← 代码是右开间
```

注释说左开间，实际代码在右开间。说明设计意图曾变化但未同步更新注释。

### 6.2 隔墙厚度不统一

| 墙 | 厚度 | 位置 |
|----|------|------|
| 外墙 | `WALL_T = 0.24` | `house-core.js:88-91` |
| 内隔墙（interiorWall） | `WALL_T * 0.8 = 0.192` | `house-core.js:99` |
| 前后隔墙（crossWall） | `WALL_T * 0.7 = 0.168` | `house-core.js:111` |

三个不同厚度值，没有统一的设计语言。
