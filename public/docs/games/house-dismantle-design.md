# House Dismantle 3D — Design Spec

> 2026-07-24 | 大别山 4 开间 2 层 (16m×9m) | 现代水泥柱民居

## 项目目标

交互式 3D 建筑模型，通过平移拆解展示民居空间布局。用户点击分类按钮将房屋按层平移，清楚观察每层每个房间的内部结构。

## 交互设计

3 个分类按钮，点击即平移，再点复位。部件列表可单独选中 → 按 1 拆解/组装。

| 按钮 | 方向 | 目的 |
|------|------|------|
| 屋顶 ↗ | +X 20m | 屋顶右移 → 看二层布局 |
| 二层 ↗ | -X 20m | 二层左移 → 看一层布局 |
| 一层 ↗ | 原地 | 一层保留原位 |

## 空间布局

```
        前 (+Z, 南) 朝南采光
  ┌───────┬───────┬───────┬───────┐
  │ 开间1 │ 开间2 │ 开间3 │ 开间4 │  ← 4m×4=16m
  │老人房 │老人房 │ 客厅  │ 厨房  │  ← 前门 (bay3 x=2)
  ├──门洞─┼──门洞─┼──门洞─┼──门洞─┤  z=0 前后隔墙
  │ 餐厅  │(楼梯) │卫生间 │ 后门  │  ← 后门 (bay4 x=6)
  │       │(开放) │       │       │  ← 楼梯从一层 bay2-3 后部上到二层
  └──走廊门洞(z≈-2)──────────────┘
        后 (-Z, 北)
```
- x=0 隔墙后部已移除（楼梯区域开放）
- x=-4, x=4 隔墙在 z=-2 处留 1m 走廊门洞
- 前后隔墙 z=0 每开间留 0.9m 门洞
- 二层布局镜像一层（主卧/次卧/书房/儿童房×2/卫生间）

## 检测规则

| 规则 | 内容 |
|------|------|
| dep-topology | 依赖关系拓扑检查 |
| room-height | 房间净高 ≥ 2.2m |
| column-placement | 柱位不挡门洞/不孤立 |
| poly-consistency | 多边形面一致性 |
| clearance | 门洞 ≥ 0.7m / 0.9m |
| overlap | 部件间重叠 > 5% |
| z-fighting | 同平面 mesh z 偏移 |
| reachability | BFS 遍历所有房间从入口可达性 |

## 待处理 (P0 — 结构)

- **等宽开间** (4×4m) — 堂屋(客厅)应与侧房不等宽，传统明间>次间>梢间
- **无独立走廊** — 走廊是隔墙上的洞，不是房间之外的独立通道空间
- **门朝向** — 内门一律朝北，未考虑风水(门不对门/不对床/不对灶)
- **楼梯出口** — 二层下楼梯后到走廊的路径不够直观
- **卫生间** — 仅蹲便+洗手盆，缺少淋浴区、排气窗

## 已知问题 (P1 — 视觉)

- **材质纹理** — 纯色无贴图，建议 canvas 程序化木纹/瓦纹/石纹
- **家具比例** — 凳子 1.6m > 桌子 1.5m 深
- **site-analyzer** — 硬编码 "后窗×4" → 实际 3 个

## 已知问题 (P2 — 代码)

- `house-yard.js` 文件已删除但 import 残留需清理
- 一层地板 (`floor`) 是否应独立于一层墙体移动 vs 和地基一起
- validate 表格输出对用户不可见 (仅 console.table)

## 设计决策记录

1. **水泥柱而非木柱** — 用户明确要求现代风格
2. **水平平移而非垂直** — 用户明确要求全部水平方向
3. **三层分类** — 用户明确只要屋顶/二层/一层三个按钮
4. **去掉院子** — 用户要求去掉鸡、井等无关元素
5. **去掉参观功能** — 之前做的 PiP 摄像头/房间面板均被用户否定
